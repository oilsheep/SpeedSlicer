# Green Key Elements — 設計規格

## 概述

一個純前端 Web App，讓遊戲開發者/美術設計師能快速將「純色背景上的 UI 素材圖」去背、自動切割成獨立元件，並匯出為帶透明度的 PNG 檔案。

## 目標用戶

遊戲開發者 / 美術設計師，自己製作素材後需要快速切圖。

## 技術棧

- 純前端：HTML + CSS + Vanilla JS
- 圖片處理：Canvas 2D API（`getImageData` / `putImageData`）
- 元件偵測：Connected Component Labeling（Two-Pass 演算法）
- 唯一外部依賴：JSZip（批次匯出 ZIP）

## 架構

4 個核心模組：

### 1. ImageLoader

- 支援拖放和檔案選擇器載入圖片
- 將圖片繪製到隱藏的 Canvas，取得 `ImageData` 像素資料
- 傳遞像素資料給後續模組

### 2. BackgroundDetector

- **自動偵測**：取圖片四角各 20x20 像素區域，統計最高頻 RGB 顏色（容差 10 內視為同色）
- **手動覆寫**：使用者可用色彩選取器指定背景色，或直接點擊圖片上任意位置取色
- 偵測結果顯示在右側控制面板

### 3. ElementSlicer

負責去背和元件切割，分為三個子功能：

#### 3a. 去背引擎

提供兩種模式，使用者可在 UI 上切換：

**漸變模式（預設）：**
- 計算每個像素與背景色的 Euclidean distance（RGB 空間）
- 色差 ≤ 內圈閾值 → alpha = 0（全透明）
- 色差 ≥ 外圈閾值 → alpha = 255（完全不透明）
- 中間 → alpha 線性插值
- Color decontamination：對半透明像素移除背景色污染，將色彩往白色方向偏移，偏移比例 = (1 - alpha/255)。效果：淺綠色元件在綠背景上會變成半透明白色
- 邊緣抗鋸齒：同閾值模式，距離使用者可調（預設 1px，範圍 0~5px）
- 使用者可調參數：內圈閾值（0~255）、外圈閾值（0~255）、抗鋸齒距離（0~5px）

**閾值模式：**
- 色差 ≤ 閾值 → alpha = 0
- 色差 > 閾值 → alpha = 255
- 邊緣抗鋸齒：對邊緣像素根據周圍透明/不透明比例混合 alpha
- 抗鋸齒距離使用者可調（預設 1px，範圍 0~5px）
- 使用者可調參數：閾值（0~255）、抗鋸齒距離（0~5px）

兩種模式皆提供即時預覽，以棋盤格背景顯示透明效果。

#### 3b. 元件偵測（CCL）

- 在去背後的像素資料上執行
- alpha > 0 的像素視為前景
- Two-Pass Connected Component Labeling 演算法掃描連通區域
- 每個連通區域產生 bounding box（最小包圍矩形）
- 過濾太小的區域（預設 < 10x10 像素），閾值使用者可調

#### 3c. 重疊/靠近偵測

- 計算所有 bounding box 之間的距離
- 距離小於閾值（預設 5px，使用者可調）的標記為「疑似重疊」
- 在預覽畫面上用不同顏色框標示
- 使用者可選擇：合併（兩框合一）或保持分開

### 4. UIController

管理整個操作流程和使用者介面。

## UI 介面

### 佈局：左大右小

- **左側**：主畫布預覽區
- **右側**：控制面板 + 元件列表
- **底部**：匯出按鈕列

### 左側 — 主畫布

- 顯示載入的圖片，去背後以棋盤格顯示透明區域
- 每個偵測到的元件用彩色矩形框標示（疑似重疊用不同顏色）
- 互動功能：
  - 縮放：滑鼠滾輪
  - 平移：拖拽空白區域
  - 切割框可拖拽移動、拖拽邊角調整大小
  - 可手動新增框（拖拽畫出新矩形）
  - 點擊框選取，對應右側元件列表高亮

### 右側 — 控制面板

分為兩個區塊：

**設定區：**
- 背景色選擇器（色塊顯示 + 色彩選取器 + 圖片取色模式）
- 去背模式切換（漸變 / 閾值）
- 對應模式的參數滑桿（內圈/外圈閾值，或閾值+抗鋸齒距離）
- 最小元件尺寸滑桿
- 重疊偵測距離滑桿

**元件列表區：**
- 顯示所有偵測到的元件
- 每項顯示：勾選框、縮圖預覽、可編輯名稱、單獨下載按鈕
- 點擊項目 → 畫布高亮對應框
- 右鍵/長按 → 刪除、與相鄰框合併

### 底部 — 匯出

- 「匯出選取的元件」按鈕（主要）
- 「全部匯出」按鈕
- 匯出 padding 設定（預設 0px）
- 批次匯出打包為 ZIP（使用 JSZip）

## 使用者操作流程

```
1. 開啟網頁
2. 拖放或選擇圖片檔案
3. 系統自動偵測背景色 → 顯示在控制面板
4. 系統自動去背 + 切割 → 畫布顯示切割框
5. 使用者預覽並調整：
   - 調整背景色（如果自動偵測不準）
   - 切換去背模式、調整參數
   - 處理重疊元件（合併/分開）
   - 拖拽調整切割框位置和大小
   - 手動新增/刪除框
   - 重新命名元件
6. 匯出為 PNG（單個或 ZIP 批次）
```

## 效能考量

- 遊戲 UI 素材圖通常 1024~2048px，Canvas 2D 處理足夠
- 參數調整時的即時預覽：使用 debounce（150ms）避免過度重繪
- 如未來需要處理更大圖片，可將去背計算搬到 Web Worker
